# 两阶段随机优化：仓库选址与路径优化

## 概述

本项目实现了一个完整的两阶段随机优化框架，专门用于解决仓库选址和车辆路径优化问题。该框架采用MATLAB编程，提供了多种算法选择和可视化功能。

## 问题描述

### 第一阶段：仓库选址决策
- **目标**：在给定的潜在仓库位置中选择最优的仓库位置
- **约束**：仓库容量限制、固定建设成本
- **决策变量**：二进制变量，表示是否在某位置建设仓库

### 第二阶段：路径优化
- **目标**：在已选定仓库的基础上，为每个随机需求场景设计最优配送路径
- **约束**：车辆容量限制、每个客户必须被服务
- **随机性**：客户需求的不确定性通过多个场景来建模

## 核心功能

### 1. 基础优化函数 (`two_stage_optimization.m`)

**主函数**: `two_stage_optimization()`

**支持参数**:
- `num_customers`: 客户数量（默认：20）
- `num_potential_warehouses`: 潜在仓库数量（默认：5）
- `num_scenarios`: 随机场景数量（默认：10）
- `warehouse_capacity`: 仓库容量（默认：1000）
- `fixed_cost_warehouse`: 仓库固定成本（默认：5000）
- `vehicle_capacity`: 车辆容量（默认：100）
- `plot_results`: 是否绘制结果（默认：true）

**算法特点**:
- 使用贪心算法进行仓库选址
- 采用最近邻启发式算法进行路径优化
- 支持多场景期望值优化

### 2. 高级优化函数 (`advanced_optimization.m`)

**主函数**: `advanced_two_stage_optimization()`

**额外支持的算法**:
- 贪心算法 (`'greedy'`)
- 遗传算法 (`'genetic'`)
- 模拟退火算法 (`'simulated_annealing'`)

**高级功能**:
- 多次运行取最优解
- 更真实的数据生成（客户聚类分布）
- 改进的VRP求解器
- 结果保存功能

## 使用方法

### 基础使用

```matlab
% 使用默认参数
[selected_warehouses, total_cost, routes] = two_stage_optimization();

% 自定义参数
[selected_warehouses, total_cost, routes] = two_stage_optimization(...
    'num_customers', 25, ...
    'num_potential_warehouses', 6, ...
    'warehouse_capacity', 1200, ...
    'plot_results', true);
```

### 高级使用

```matlab
% 使用遗传算法
[best_solution, all_solutions] = advanced_two_stage_optimization(...
    'algorithm', 'genetic', ...
    'num_runs', 10, ...
    'max_iterations', 200);

% 使用模拟退火算法
[best_solution, all_solutions] = advanced_two_stage_optimization(...
    'algorithm', 'simulated_annealing', ...
    'num_runs', 5, ...
    'save_results', true);
```

### 运行示例

```matlab
% 运行所有示例
run('example_usage.m');
```

## 算法说明

### 1. 贪心算法
- **思路**：枚举所有可能的仓库组合，选择总成本最低的组合
- **优点**：计算速度快，适合小规模问题
- **缺点**：可能无法找到全局最优解

### 2. 遗传算法
- **思路**：使用二进制编码表示仓库选择，通过交叉和变异操作进化种群
- **优点**：全局搜索能力强，适合复杂问题
- **缺点**：计算时间较长，需要调参

### 3. 模拟退火算法
- **思路**：从初始解开始，通过邻域搜索和概率接受机制逐步改进解
- **优点**：能跳出局部最优，参数较少
- **缺点**：收敛速度依赖于温度调度策略

## 参数调优建议

### 1. 问题规模设置
- **小规模问题**：customers ≤ 20, warehouses ≤ 5, scenarios ≤ 10
- **中等规模问题**：customers 20-50, warehouses 5-10, scenarios 10-20
- **大规模问题**：customers > 50, warehouses > 10, scenarios > 20

### 2. 算法选择
- **快速求解**：使用贪心算法
- **高质量解**：使用遗传算法或模拟退火
- **不确定问题规模**：先用贪心算法测试，再选择合适的高级算法

### 3. 容量设置
- 仓库容量应大于平均总需求的1.2-1.5倍
- 车辆容量建议设为单个客户最大需求的2-5倍

## 常见问题

### Q1: 程序运行时间过长
**解决方案**：
- 减少场景数量或客户数量
- 使用贪心算法而非遗传算法
- 设置`plot_results = false`关闭绘图

### Q2: 某些客户未被服务
**解决方案**：
- 增加车辆容量
- 增加仓库数量
- 检查需求设置是否合理

### Q3: 结果不稳定
**解决方案**：
- 增加运行次数(`num_runs`)
- 使用固定随机种子
- 调整算法参数
